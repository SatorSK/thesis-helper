# PRD — 국제통상 졸업논문 작성 지원 도구 (Thesis Helper)

> **버전** v0.2 · **작성일** 2026-06-08 · **대상** 동국대 국제통상학과 졸업논문
> **한 줄 정의:** 단락별 초안·자료·인용을 모듈로 생성하되, **본문은 사용자가 자기 문장으로 재작성**하도록 설계된 로컬 Streamlit 앱. 표절·AI탐지 우회 도구가 **아니라**, 원본성을 정공법으로 끌어올리는 도구.

---

## 0. 핵심 원칙 (Non-negotiables)

이 도구가 절대 하지 않는 것 / 반드시 하는 것. 모든 모듈 설계는 이 원칙에 종속된다.

| 구분 | 원칙 |
|------|------|
| ❌ 안 함 | 표절검사기(카피킬러)·AI탐지기(GPTKiller 등) **우회·회피 기능** |
| ❌ 안 함 | "사람처럼 보이게" 문장 변조, 동의어 치환식 패러프레이징 자동화 |
| ❌ 안 함 | 출처 없는 본문 자동 완성·납품 |
| ✅ 함 | 단락 **초안**을 주고, 사용자가 **자기 문장으로 재작성**하도록 강제 |
| ✅ 함 | 모든 차용 문장에 **출처·인용 자동 부착** (표절률을 낮추는 정공법) |
| ✅ 함 | 본인 데이터·실증분석으로 **독창적 기여**를 만들게 지원 (표절률 0 구간 확대) |
| ✅ 함 | 어떤 단락이 "남의 생각 요약"인지 표시해 **인용 누락을 사용자가 직접 메우게** 함 |

**작업 분담 합의:** 도구 = 골격·자료·단락초안·인용. 사용자 = 모든 본문의 최종 문장, 논리 검증, 마무리.

## 0.5 동국대 국제통상학과 졸업논문 규정 (검증됨)

> 출처: 국제통상학과 졸업논문 제출 안내 공지(2024학년도 기준). **2026 최신 공지 재확인 필요(§8).**

| 항목 | 규정 | 도구 반영 |
|------|------|-----------|
| 용지·여백 | A4 / 상하좌우 **30mm**, 머리말·꼬리말 15mm | M7 .docx 자동 적용 |
| 글자·줄간격 | 본문 **11pt**, 주석 **9pt**, 줄간격 **160%** | M7 자동 적용 |
| 분량 | 본문 **20장 이상**(표지·목차·참고문헌 제외) | M3·M5에 분량 게이지 |
| 표지 | 제목·학과·학번·지도교수명(복수전공 제외)·이름·제출일 | M7 표지 자동 생성 |
| **인용 방식** | **각주(footnote) 필수** — "인용 시 각주를 필히 달 것" | **M2·M5 각주 우선**, 참고문헌은 APA식 목록 |
| **표절률 기준** | 카피킬러 **30% 이하** (교육부 6어절/1문장 기준) | M6 자가점검의 안전 목표선 |
| 제출물 | 논문 1부 + **카피킬러 결과보고서** | 사용자가 별도 카피킬러 돌려 첨부 |
| 제출 | 이메일 itrade@dongguk.edu, 파일명 `(국제통상학과)학번_이름` | M7 파일명 규칙 자동 |

**인용 방식 결정:** 학과 규정(각주)이 사용자 선호(APA)보다 우선. → **본문 인용 = 각주 / 참고문헌 목록 = APA 7th 형식**으로 절충. M2·M5는 각주 텍스트를 1차 산출, APA는 목록용.

## 1. 목표 (Goals)

1. **참신한 주제**를 데이터 확보 가능성 기준으로 골라준다.
2. **선행연구·1차 자료**를 모아 APA로 정리한다 (인용 누락 = 표절률의 주범).
3. 혼합형(정성+보조데이터) 논문의 **표준 골격**과 단락별 초안을 제공한다.
4. 본인 **데이터 분석**(무역지표·통계)으로 독창적 분석 단락을 만들게 돕는다.
5. 완성 전 **자가 점검**: 인용 누락, AI 작성 의심 단락, 자기표절 위험을 사용자에게 표시.

### Non-Goals
- 자동 제출·자동 완성 논문 생성
- 탐지기 점수 예측/우회
- 교수·심사 커뮤니케이션 대행

## 2. 사용자 & 사용 흐름

**사용자:** 동국대 국제통상학과 학부생 (본인). 코딩 비전공 가정 → 버튼·폼 위주 UI.

**전체 흐름 (좌측 사이드바 = 모듈 = 논문 단계):**

```
① 주제 발굴 → ② 자료수집·인용관리 → ③ 골격 빌더 → ④ 데이터 분석
   → ⑤ 단락 초안 작성소 → ⑥ 자가 점검(인용/AI/자기표절) → ⑦ 내보내기(.docx)
```

각 모듈은 독립 실행 가능(모듈화)하지만, 공통 프로젝트 파일(`project.json`)을 읽고/쓴다.

## 3. 모듈 명세 (MVP)

### M1. 주제 발굴 (Topic Finder)
- **입력:** 관심 키워드(예: CBAM, 공급망, 디지털무역), 데이터 접근 가능성 우선순위.
- **처리:** 최신 국제통상 이슈 풀에서 후보 주제 생성 → 각 후보에 ⓐ참신성 ⓑ데이터 확보 난이도 ⓒ선행연구 충분성 점수 부여.
- **출력:** 주제 후보 10개 + 각 후보의 핵심 연구질문(RQ) 초안 + 추천 데이터 출처.
- **재작성 규율:** RQ는 초안일 뿐, 사용자가 확정·수정.

### M2. 자료수집 · 인용관리 (Source & Citation Manager)
- **입력:** 논문/보고서 제목·DOI·URL 또는 직접 입력.
- **처리:** 메타데이터 정리 → 인용 ID 부여(`[hong2024]`) → ⓐ **각주 텍스트**(본문용, 학과 규정) + ⓑ **APA 7th 참고문헌 항목**(목록용) 동시 생성.
- **출력 예:** 각주 `홍길동(2024), 「제목」, 학술지, 12(1), 34쪽.` / 참고문헌 `홍길동. (2024). 제목. 학술지, 12(1), 30–45.`
- **데이터 출처 후보:** RISS, DBpia, Google Scholar, WTO, UN Comtrade, 한국무역협회(KITA), KOTRA, 한국은행 ECOS. *(자동 크롤링은 단계적; MVP는 수동 입력 + 형식 자동화부터)*
- **핵심:** 모든 차용 아이디어에 인용 ID를 달아두면 M5에서 본문에 자동 삽입 → 인용 누락 방지.

### M3. 골격 빌더 (Outline Builder)
- **혼합형 표준 골격:** 서론 → 선행연구 검토 → 분석틀/연구방법 → 정성분석(사례·정책) → 보조 데이터분석 → 논의 → 결론·시사점.
- **출력:** 절·소절 구조 + 각 절의 목적 1줄 + 필요한 자료/데이터 체크리스트.
- 사용자가 절을 추가/삭제/재배열.

### M4. 데이터 분석 (Data Analysis) — 혼합형 보조
- **입력:** CSV 업로드(무역통계 등) 또는 KITA/Comtrade에서 받은 표.
- **처리:** 기초 통계, 추세 그래프, 간단한 상관/비교(학부 수준). 표·그림 자동 캡션(APA).
- **출력:** 그래프 PNG + 표 + "이 그래프가 말하는 것" **데이터 사실 요약**(해석은 사용자 몫).
- **원본성 핵심:** 본인이 돌린 분석 = 표절률 0. 가장 강한 모듈.

### M5. 단락 초안 작성소 (Paragraph Drafter) — ⚠️ 핵심 규율
- **입력:** 절 선택 + 관련 인용 ID(M2) + 데이터 결과(M4).
- **처리:** 해당 단락의 **초안**과 **들어가야 할 논점 체크리스트** 생성. 차용 논점엔 인용 ID 자동 부착.
- **UI 강제 장치:**
  - 좌: 초안(읽기 전용, "참고용 — 복붙 금지" 라벨)
  - 우: **빈 입력칸 = 사용자가 자기 문장으로 작성**
  - 초안 텍스트는 최종 .docx로 **내보내지지 않음**. 우측 사용자 작성분만 내보내짐.
  - "초안 그대로 복사" 감지 시 경고(유사도 높으면 내보내기 차단 옵션).
- **목적:** 도구는 생각의 비계(scaffold), 문장은 사람.

### M6. 자가 점검 (Integrity Self-Check)
- **인용 누락 점검:** 출처 없이 사실·수치·주장을 담은 문장 표시 → 사용자가 인용 추가하도록.
- **AI 작성 의심 점검:** 자기 글에서 단조로운 패턴·일반론 문장을 표시 → 사용자가 구체화/재작성하도록 (*탐지기 점수 예측이 아니라, 글을 더 본인 것으로 만들라는 신호*).
- **자기표절 점검:** 본인 과제·이전 글 재사용 여부 사용자 체크리스트.
- **출력:** 단락별 "보강 필요" 리스트. 점수가 아니라 **할 일 목록**.

### M7. 내보내기 (Export) — 동국대 양식 자동 적용
- M5의 **사용자 작성분 + 각주 + M2 참고문헌 + M4 표/그림** → .docx.
- **학과 규정 자동 적용:** A4·여백 30mm·머리말 15mm·본문 11pt·주석 9pt·줄간격 160%·본문 20장 분량 게이지.
- **표지 자동 생성:** 제목·학과·학번·지도교수명·이름·제출일 입력 폼 → 표지 페이지.
- **인용 = 각주**로 삽입(학과 규정), 참고문헌 목록은 APA식.
- **파일명 자동:** `(국제통상학과)학번_이름.docx`.
- 카피킬러 결과보고서는 사용자가 카피킬러에서 직접 발급해 첨부(도구는 안내만).

## 4. 기술 스택

| 영역 | 선택 | 비고 |
|------|------|------|
| UI | Streamlit | 비개발자용 폼·버튼 |
| 언어 | Python 3.11+ | |
| 저장 | 로컬 `project.json` + `/sources`, `/data`, `/figures` | 클라우드 X, 로컬 보관 |
| 인용 | APA 7th 포맷터 (자체 또는 `pybtex`) | |
| 데이터 | pandas, matplotlib | 학부 수준 분석 |
| 문서 | python-docx | .docx 내보내기 (학과 양식 자동 적용) |
| LLM(초안용) | **provider-agnostic 어댑터** | 아래 참조. 키 없으면 템플릿 모드 |

### LLM 백엔드 — 교체 가능 설계 (provider-agnostic)
초안 생성(M1·M5)은 특정 회사에 묶지 않는다. `core/llm.py`에 **단일 인터페이스** `complete(prompt, system) -> text`를 두고, 어댑터로 갈아끼운다.

| 백엔드 | 환경변수 | 비고 |
|--------|----------|------|
| OpenAI / Codex | `OPENAI_API_KEY` | GPT 계열, OpenAI 호환 엔드포인트 |
| Claude | `ANTHROPIC_API_KEY` | |
| Google Gemini | `GOOGLE_API_KEY` | |
| OpenAI-호환 로컬(예: Ollama, LM Studio) | `OPENAI_BASE_URL` 재지정 | 로컬 모델 |
| **템플릿 모드(키 없음)** | — | LLM 없이 골격·체크리스트·각주 형식만. **기본값** |

- UI 사이드바에서 **백엔드 선택 + 키 입력**(세션에만 보관, 디스크 저장 안 함).
- **로컬 우선·외부전송 최소화:** 논문 원고 전체를 절대 통째로 보내지 않음. M5는 *해당 단락의 논점·인용만* 전송. LLM 끄면 전송 0.

## 8. 결정 사항 & 남은 확인 (2026-06-08 업데이트)

**결정됨:**
1. **LLM 백엔드** → provider-agnostic. 기본은 템플릿 모드(키 없음), OpenAI/Codex·Claude·Gemini·로컬 어댑터 교체 가능. (사용자: Codex/기타 API 사용)
2. **논문 양식 파일** → 별도 파일 없음 → 학과 공지 규정값(여백 30mm·11pt·160%·각주·20장)을 M7에 하드코딩.
3. **주제** → 정해진 것 없음 → M1으로 0부터 발굴.
4. **데이터 범위** → 미정 → 도구가 추천(아래 §8.1). M1 주제 확정 시 동기화.

**남은 확인(사용자 액션, 차단 요소 아님):**
- **동국대 2026 최신 공지 재확인** — 위 규정은 2024 기준. 30% 기준·각주 규정·분량이 바뀌었을 수 있음. itrade@dongguk.edu 또는 학과 공지에서 최신 졸업논문 안내 확인 후 알려주면 M7에 반영.
- **AI 작성 탐지 적용 여부** — 공지엔 카피킬러(유사도)만 명시, AI탐지기 별도 적용 여부는 미확인. (도구는 어느 쪽이든 "본인이 쓴 글" 원칙이라 영향 없음)

### 8.1 데이터 분석 범위 — 도구 추천(혼합형 학부논문 기준)
M1 주제와 동기화하되, 데이터 확보·분석 난이도를 고려한 기본 권장:
- **대상국:** 한국 중심 + 비교 1~2개국/블록(예: EU, 미국, 중국, 베트남 중 주제별 택).
- **품목:** HS 2~4단위 **특정 품목군 1개**(예: 철강 HS72, 반도체 HS8541, 이차전지 소재). 전체 무역 말고 좁혀야 분석이 선명.
- **기간:** **2015~2024(10년)** — COVID·공급망 재편·보호무역 충격을 모두 포함.
- **출처:** UN Comtrade(품목별 양자 무역), KITA 무역통계, 한국은행 ECOS(환율·교역조건). 모두 공개·합법.
- **분석 수준(학부):** 추세·점유율 변화·간단 비교, 필요시 단순 회귀 1개. 고급 계량은 비권장(검증 부담).
